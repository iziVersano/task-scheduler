## Databases and EC2 Instances
- **Real-Time Processing**: Defined as the capability to handle multiple simultaneous operations and requests efficiently.
- **Database Connections**: EC2 instances connect to databases to fetch/store data in a two-way direction. 
    - EC2 must have adequate RAM as a temporary cache to minimize latency and relieve database load.
    - High traffic to databases risks overloading servers and network bandwidth, leading to performance bottlenecks.
- **Caching Strategy**: RAM is used to temporarily store frequently accessed data (e.g. product details) to avoid repeated database queries, reduce latency, and prevent network clutter.

> 🏢 Real world: E-commerce websites like Amazon and Zalando use caching for product pages to reduce the waiting time for customers accessing popular items simultaneously.

## Virtual Private Cloud (VPC)
- **Key Characteristics**:
    - **Virtual**: Accessible over the internet and isolated from the broader internet.
    - **Private**: Exclusive to the organization that creates it; not public-facing.
    - **Cloud-based**: Allocated within the AWS infrastructure.
- **Purpose**: VPCs are necessary to define the scope and availability of IP addresses for AWS services such as EC2, S3, RDS, etc.
- **Default VPC**:
    - Every AWS region has one pre-configured default VPC.
    - Default VPCs are primarily for learning purposes, not production environments.
- **Custom VPC Setup**:
    - Businesses can create custom VPCs for specific resource allocation and security needs.
    - VPCs use CIDR blocks (e.g., `/16`) to define the number of available IP addresses — e.g., `/16` offers approximately 65,536 IPs.

| Feature            | Default VPC          | Custom VPC           |
|--------------------|----------------------|----------------------|
| Availability       | In every region      | Created by the user  |
| Usage Purpose      | Learning only        | Production-ready     |
| IP Allocations     | Predefined by AWS    | Customizable by user |

> 🏢 Real world: Each company utilizing AWS (e.g., Netflix or Airbnb) creates custom VPCs to segment resources securely and optimize resource usage according to their needs.

## Instance Types
- **Categories**:
    - **Compute-Optimized**: Ideal for tasks requiring high CPU output.
    - **Memory-Optimized**: Designed for applications involving frequent data access and manipulation in memory (e.g., interacting with databases or caching for temporary storage).
    - **Network/Storage-Optimized**: Suited for massive data analytics or high-storage applications like warehouses and extensive customer databases.
- **Example Classes**:
    - Memory Optimized: *R5, R5a, R5b.*
    - Storage Optimized: *D2, D2 Extra Large.* (e.g., 244 GB of RAM, $6.50/hour deployment costs).
- **Use Cases**: Choosing the correct instance type is critical according to business requirements like database operations, RAM needs, read/write performance, or data warehousing.

| Instance Type       | Recommended Use Case             | Example Classes             |
|---------------------|----------------------------------|-----------------------------|
| Compute-Optimized   | High CPU demand tasks           | N/A                         |
| Memory-Optimized    | Database-intensive tasks        | R5, R5a, R5b               |
| Network/Storage Optimized | Warehousing, large datasets    | D2, D2 Extra Large          |

> 🏢 Real world: Supermarket chains like Aldi and Netto utilize storage-optimized instances to maintain large-scale data warehouses for product inventory management (exceeding 100 TB). High memory and large storage instances are essential for fast reads/writes and real-time data processing.

## Additional Notes
- **Default VPCs**: Standard resource in every region. Beginners often start with EC2 inside a default VPC.
- **Hypervisors**: EC2 instances use virtualization (e.g., VMware VirtualBox) in the backend managed by AWS, simulating several operating systems on a single server.
- **IP Addresses**: Every AWS service, whether server-based or serverless (e.g., Lambda), has an IP address in the backend for routing.

> 🏢 Real world: Cloud-native companies utilize default VPCs for testing and training purposes and switch to custom VPCs for production workloads.

## Exam Tips
1. **Instance Types** and Specifications:
   - Memory Optimized for high-performance databases.
   - Storage Optimized for large-scale data warehouses.
2. **VPC**:
   - Every AWS region has a default VPC—exam question answer: True.
   - CIDR block size determines IP address availability: e.g., `/16` equates to 65,536 IPs.
3. **Networking**:
   - Always use custom VPCs for production.
   - Each AWS service has a dedicated IP in the backend.
4. **Caching for Reducing Latency**:
   - Effective caching requires proper utilization of temporary memory (RAM).
5. **Resource Allocation for VPC Services**:
   - Plan the usage of IPs for various services (e.g., allocate different IPs for EC2, RDS, etc.).
