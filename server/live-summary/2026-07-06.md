## AWS Access Management
- **Access Key & Secret Access Key**:
  - Access key acts as the username and secret access key acts as the password for authenticating AWS services.
  - Storing credentials securely is crucial; never hard code sensitive information into scripts.
  - If credentials are compromised, revoke and regenerate them immediately.
  
- **How to Use Credentials**:
  - Credentials can be accessed via text files, Excel sheets, or through secure storage methods.
  - Recommended to utilize tools or environment variables to avoid hard coding values.
  - AWS provides the credentials only once after creation.

- **Access Methods**:
  - AWS services can be accessed via CLI, Console, and SDK depending on the requirements.

> 🏢 Real world: **Capital One** uses access keys for automating AWS CloudFormation stacks via CLI while ensuring credentials management follows strict security protocols.

## Terraform Basics for AWS
- **Cloud Agnostic Nature of Terraform**:
  - Terraform works across multiple cloud providers (not specific to AWS).
  - Requires a `provider` block in `main.tf` to specify which cloud platform resources need to be created on.

- **Provider Block**:
  ```plaintext
  provider "aws" {
    region = "eu-central-1" # Specify the region
  }
  ```
  - The provider block specifies the cloud platform (e.g., AWS) and its configuration.
  - First block in `main.tf` typically is your `provider`.
  
- **Hands-On Code Writing**:
  - Major disadvantage: Requires manual code writing.
  - Enable AI tools like GitHub Copilot in code editors for automated assistance.

> 🏢 Real world: **Netflix** leverages Terraform to manage resources efficiently across multiple AWS regions while maintaining reusable infrastructure code.

## Creating S3 Buckets with Terraform
- **S3 Bucket Basics**:
  - Bucket names must be globally unique.
  - Bucket naming rules:
    - Length: Between 3 to 63 characters.
    - Avoid special characters, uppercase letters, or spaces.
  
- **S3 Terraform Example Code**:
  ```hcl
  provider "aws" {
    region = "eu-central-1"
    access_key = "ACCESS_KEY"
    secret_key = "SECRET_KEY"
  }

  resource "aws_s3_bucket" "example_bucket" {
    bucket = "unique-bucket-name"
    tags = {
      department = "aws"
      date       = "2026-07-06"
    }
  }
  ```

- **Logical Name**:
  - Logical IDs (e.g., `"example_bucket"`) are used for referencing within Terraform templates but cannot be accessed outside Terraform.

- **Documentation Advantage**:
  - Always refer to AWS Terraform Registry for accurate syntax and examples.
  - Example usage codes can save time and reduce errors.

> 🏢 Real world: **Dropbox** optimized its storage operations using S3 bucket provisioning with Terraform, ensuring compliance and scalability of storage solutions across global regions.
