## EC2 Instance Lifecycle

- **States Explained**:
  - **Start**: When starting an instance from a stopped state, a public IP is reassigned.
  - **Stop**: 
    - CPU stops.
    - RAM is released.
    - Public IP address is removed.
  - **Terminate**: Permanently deletes the instance and detaches associated resources, such as storage.

- **IP Address Behavior**:
  - Public IP changes every time an instance is stopped and restarted.
  - Instances lose their public IP upon stopping, which can cause complications when associated with domain names.

> 🏢 **Real world:** A company like **Google** stops its EC2 instances during low-traffic hours to save costs. However, since starting the instance generates a new public IP, websites such as `google.com` that are linked to a static IP may experience downtime (e.g., 404 errors) unless mitigated by using Elastic IP.

## Elastic IP (EIP)

- **Definition**: A static, public IPv4 address that you can allocate and attach to an EC2 instance.
- **Key Benefit**: Addresses the issue of dynamic public IP reassignment when an instance stops or restarts.
- **IP Assignment**:
  - EIP maintains a fixed IP for an instance regardless of its state (running, stopped, or hibernated).
  - This ensures consistent DNS resolution for domain names linked to the instance.

| Public IP        | Elastic IP           |
|-------------------|----------------------|
| Assigned dynamically at start. | Statically assigned to the instance. |
| Changes when the instance is stopped/restarted. | Remains the same irrespective of the instance state. |
| Free, but ephemeral. | Free if associated with running instance; charges if not attached. |

> 🏢 **Real world:** A company experiencing user complaints about intermittent access due to IP changes assigns an **Elastic IP** to their backend EC2 instances to guarantee consistent user access.

## DNS & Domain Names

- **What is a domain name?**
  - A human-readable name (e.g., `google.com`) that is mapped to an IP address.
  - Domain names are registered via services like **Route 53**.
  - Requires payment for purchasing and registration.
  
- **DNS (Domain Name System)**:
  - Translates domain names to their respective IP addresses.
  - If an instance's IP changes, the domain name linked to the old IP will fail to resolve (e.g., return a 404 error).

> 🏢 **Real world:** A company using **Route 53** registers a custom domain and links it to their EC2 instances. This ensures their users can easily remember and access their services without needing to know the underlying IP addresses.

## Security Groups

- **Definition**: A virtual firewall that controls inbound and outbound traffic for EC2 instances.
- **Key points**:
  - **Region-bound**: Security groups are region-specific and not tied to availability zones.
  - **Multiple Attachments**: A security group can be linked to multiple EC2 instances.
  - **Default State**:
    - All **Inbound** traffic is **Blocked** (no incoming connections allowed by default).
    - All **Outbound** traffic is **Allowed** (instance-originating requests are allowed by default).
  - **Rules**:
    - Inbound and outbound traffic must be explicitly defined using rules.
    - Common ports include:
      - Port 22: SSH (Secure Shell)
      - Port 80: HTTP
      - Port 443: HTTPS
      - Port 21: FTP (File Transfer Protocol)

> 🏢 **Real world:** A company hosting a web server on EC2 configures its **security group** to allow inbound HTTP (port 80) and HTTPS (port 443) traffic. Additionally, developers configure SSH (port 22) access for management while blocking unnecessary access for security.
