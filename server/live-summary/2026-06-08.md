## Users and User Groups in AWS IAM
- **User Limits**:
  - AWS does not impose a maximum limit on the number of users that can be created.
  - Recommendation: Keep the number of users manageable (~50-60 users for admin efficiency).
  - Inactive users (e.g., former employees) should be deleted to maintain an optimized user directory.
  
- **User Groups**:
  - Example: A group named "AWS Cloud AWS One" contains 16 users.
  - Groups allow centralized management of permissions for multiple users.
  - Logged-in activity and last activity timestamps per user can be tracked.

> 🏢 Real world: A midsized software company organizes its developers into groups like "Frontend Engineers" or "Data Engineers" for easier permissions management.

---

## Permission Management in AWS IAM
- **Permissions Types**:
  - Standard Permissions: Predefined by AWS based on common usage patterns.
  - Custom Permissions: User-defined based on specific organization needs, often created in **JSON format**.

- **Policy Options**:
  - **Attach Policy**: Use an existing AWS-managed policy.
  - **Create Inline Policy**: Write a custom policy (JSON key-value pairs).
  
- **Key Permissions Concepts**:
  - Full Access: Enables all actions (e.g., read, write, update) on a resource.
  - Read-Only Access: Allows viewing but no modification.
  - Policies are mapped to services (e.g., EC2 Full Access, EC2 Read-Only).

| **Policy Option**    | **Description**                          | **Sample Use Case**                             |
|-----------------------|------------------------------------------|------------------------------------------------|
| Attach Policy         | Apply AWS-managed prebuilt policies     | Grant read-only access to new interns          |
| Create Inline Policy  | Create custom JSON-based rules          | Restrict a user to only start/stop EC2 instances |

> 🏢 Real world: An e-commerce company gives developers Read-Only access to production servers while administrators maintain Full Access.

## JSON in AWS Policy Configuration
- **Definition and Format**:
  - JSON (JavaScript Object Notation) is used to write custom AWS policies.
  - JSON data is structured using **key-value pairs**.
    - Example: `{ "Name": "Sharanya", "Age": 25, "Role": "Admin" }`
  - Common in other cloud automation tools like **CloudFormation** and **Terraform**.

> 🏢 Real world: A fintech startup uses custom JSON IAM policies to limit developers' access to specific services while restricting sensitive financial data.

## Role of IAM in EC2 Access
- **Restricting/Granting Access**:
  - Default: New users may have zero permissions, resulting in "Access Denied" errors.
  - Change permissions by attaching specific policies to a user's IAM group.
    - Example: Providing EC2 Full Access to enable all actions within AWS EC2.

- **Testing Permissions**:
  - User permissions can be tested live via the AWS Console. For example:
    - Without proper EC2 permissions, users cannot interact with EC2 instances.
    - Attaching the "AmazonEC2FullAccess" policy enables access to EC2 resources.

- **Permission Removal**:
  - Admins (root users) can remove permissions after a session to revoke access.

> 🏢 Real world: A multi-national retail firm restricts access to sensitive resources, giving temporary permissions to vendors or contractors for specific tasks.

## AWS Health Dashboard
- AWS Health Dashboard provides service health information for resources.
- By default, restricted access IAM roles cannot access this dashboard.
- Useful for monitoring infrastructure during events like autoscaling or the setup of load balancers.

> 🏢 Real world: A cloud management team in a global organization uses AWS Health Dashboard to monitor and address potential issues with production services deployment.

## Infrastructure as Code (IaC) and Terraform Introduction
- **Definition**:
  - IaC automates infrastructure setup using declarative configuration files.
  - Tools like **Terraform** use code to manage resources, offering repeatability and version control.

- **Key Concept**:
  - Replaces manual console-driven actions with scripts/code.
  
- **Relevance of JSON**:
  - Terraform uses JSON (key-value) format for configuration files.

> 🏢 Real world: A SaaS platform uses Terraform for rapid deployment of test environments in AWS for its QA automation pipeline.

## AWS Root User vs IAM User
- **Root User**:
  - Has full administrative access.
  - Can create/delete other IAM users.

- **IAM User**:
  - Can only have permissions explicitly provided by the root user or an administrator with the "AdministratorAccess" policy.
  - Cannot create additional users unless granted AdministratorAccess.

> 🏢 Real world: A marketing agency ensures only their IT admin team has root user privileges to manage Penguin website hosting on AWS.
