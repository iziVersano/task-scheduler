## Elastic Block Store (EBS)
- **Key Features:**
  - EBS volumes are durable, persistent block storage used with EC2 instances.
  - Elastic nature: EBS volumes can be resized for adjustments in size, type, or volume configurations. Additional steps might be needed in the operating system.
  - Snapshots: Point-in-time backups of EBS volumes.
    - Initial snapshot captures full volume; successive snapshots capture only changes.
    - Snapshots can be used for disaster recovery by automating cross-region copies.
    - Cross-region snapshots aid in regional failure management.
- **Use Cases:**
  - Golden AMI Strategy:
    - Create an AMI from a snapshot with preinstalled configurations and updates.
    - Perform rolling updates by launching instances from a golden AMI.
    - Different AMIs for development (e.g., v1.1) and production (e.g., v1.0).
  - Encryption:
    - Ensures compliance and security.
    - Even root volumes can be encrypted.
    - Changing encryption settings requires creating a new volume, selecting “encrypted,” and defining a key.
  - Types of EBS:
    - General Purpose SSD (Gp3): Suitable for most workloads.
    - IO2: High-performance but ~15x costlier than Gp3.

> 🏢 **Real World: Netflix uses EBS snapshots and AMIs for consistent and scalable EC2 deployments across regions. This supports high uptime for its streaming services worldwide.**

## Amazon Machine Images (AMIs)
- **Key Concepts:**
  - Snapshots can be used to create AMIs.
  - AMIs can include pre-installed software, configurations, and preloaded scripts.
  - Launching EC2 instances from AMIs avoids repetitive software installation/setup.
  - AMIs are regional by nature but can be copied to other regions for global use.
  - Versioning with AMIs is critical for production and development environments.
    - Update development AMI weekly.
    - Perform well-tested updates to production AMI.

- **Golden AMI Approach vs Dynamic Configuration:**
| **Aspect**            | **Golden AMI**                       | **Dynamic Configuration**       |
|-----------------------|---------------------------------------|----------------------------------|
| **Use Case**          | Preloaded image, less setup time     | On-the-fly configuration         |
| **Updates**           | Create new AMI for updates           | One-time setup, dynamic changes  |
| **Time Efficiency**   | Faster for deployments               | May take longer for setup        |
| **Production Cycles** | No need to use production cycles     | Updates may cause downtime       |

> 🏢 **Real World: Spotify applies the Golden AMI approach for their servers, allowing quick deployment of pre-configured applications while reducing setup time for scalability.**

## Snapshot Management and Disaster Recovery
- **Features of Snapshots:**
  - Serve as point-in-time backups of EBS volumes.
  - Used to create AMIs or restore to a specific state.
  - Only stores incremental changes after the first snapshot.
  - Can be triggered manually or via automation tools like Lifecycle Manager.
- **Cross-Region Snapshots**
  - Necessary for disaster recovery and business continuity.
  - Use "copy snapshot" to replicate snapshots to another region.
  - Snapshots are region-specific when created manually or via automation.
- **Usage Scenarios:**
  - Create snapshots during non-peak hours to optimize performance.
  - Automate routine snapshots with Lifecycle Manager.

## Encryption and Security for EBS
- **Key Concepts:**
  - Encryption protects data at rest and during snapshot-based backups.
  - Root EBS volumes and additional EBS volumes can all be encrypted.
  - Encryption settings can be updated by creating a new encrypted volume.
  - Use AWS-managed or customer-managed encryption keys.
  
- **Steps to Encrypt an Unencrypted Volume:**
  1. Create a new EBS volume with encryption enabled.
  2. Migrate data from the unencrypted EBS volume to the encrypted one.
  3. Detach the original volume and attach the encrypted volume to the EC2 instance.

> 🏢 **Real World: Capital One secures customer data in AWS by enabling encryption for all EBS volumes, ensuring compliance with financial regulations such as PCI DSS.**

## Tools and Automation
- **Image Builder:**
  - Automates the creation, patching, and testing of AMIs.
  - Supports weekly Golden AMI updates.
  - Reduces manual errors and repetitive tasks by streamlining AMI lifecycle management.
  
- **Lifecycle Manager:**
  - Automates EBS snapshot creation and retention policies.
  - Enables efficient snapshot storage management and cross-region backups.

> 🏢 **Real World: Atlassian uses Image Builder and Lifecycle Manager to streamline environment provisioning, ensuring up-to-date AMIs and routine backups for Jira Cloud.**

## General EBS Best Practices
- Stop EC2 instances before creating snapshots for consistency.
- Use automated tools like Lifecycle Manager for routine operations.
- Ensure proper allocation of EBS types (e.g., Gp3 for general, IO2 for high-performance).
- Monitor snapshot storage usage to optimize costs.
  
> 🏢 **Real World: Airbnb monitors EBS storage usage and optimizes costs by leveraging automated snapshot lifecycles and resizing EBS volumes based on workload patterns.**
